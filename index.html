Sync su Supabase (opzionale, con RLS)

Ogni dispositivo si autentica in modo anonimo (via Supabase Auth) e vede solo la propria riga, protetta da Row Level Security — sicuro anche con repo GitHub pubblico. SQL da eseguire una volta nel progetto Supabase: 

create table portfolio_dashboard (
  user_id uuid primary key references auth.users(id) on delete cascade,
  data jsonb not null,
  updated_at timestamptz not null default now()
);
alter table portfolio_dashboard enable row level security;
create policy "select_own" on portfolio_dashboard for select using (auth.uid() = user_id);
create policy "insert_own" on portfolio_dashboard for insert with check (auth.uid() = user_id);
create policy "update_own" on portfolio_dashboard for update using (auth.uid() = user_id) with check (auth.uid() = user_id);

Serve anche attivare "Anonymous sign-ins" in Authentication → Sign In / Providers nel dashboard Supabase.

Collegare un altro dispositivo: genera qui il token su questo dispositivo, poi incollalo nelle Impostazioni dell'altro dispositivo. È un segreto personale: non condividerlo e non salvarlo nel codice.
